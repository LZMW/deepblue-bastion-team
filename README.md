## 📋 团队简介

DeepBlue Bastion（深蓝堡垒）是一个专业的代码审查和质量保证团队，通过多专家协作，为代码库提供全面的质量分析、安全审计、架构评估和性能优化服务。

### 核心能力

- 🔍 **代码审查**：全面的代码质量分析
- 🛡️ **安全审计**：OWASP Top 10漏洞检测
- 🏗️ **架构评估**：系统架构和模块依赖分析
- ⚡ **性能优化**：性能瓶颈识别和优化建议
- 🔧 **代码简化**：复杂度降低和重构建议
- 🧪 **测试设计**：全面的测试用例设计
- 📊 **可行性评估**：技术选型和成本效益分析

## 🔄 执行模式

DeepBlue Bastion 是混合型团队，根据任务特点智能选择执行模式：

### 串行模式
- **适用场景**：任务有强依赖关系
- **执行方式**：阶段1 → 阶段2 → 阶段3
- **典型任务**：架构分析 → 安全审计 → 性能优化

### 并行模式
- **适用场景**：任务相互独立
- **执行方式**：专家1 ∥ 专家2 ∥ 专家3
- **典型任务**：多维度代码审查

### 混合模式
- **适用场景**：任务部分串行、部分并行
- **执行方式**：阶段1 → (阶段2 ∥ 阶段3)
- **典型任务**：架构分析 → 并行审计（安全/性能/复杂度）

## 📖 最佳实践

### 1. 明确任务目标
在触发团队前，尽量明确：
- 审查范围（整个项目/特定模块）
- 关注重点（安全/性能/架构）
- 输出格式（报告/代码建议）

### 2. 选择合适的执行模式
- 简单任务：使用并行模式（快速）
- 复杂任务：使用混合模式（全面）
- 依赖关系强的任务：使用串行模式（有序）

### 3. 充分利用MCP工具
在需要专业建议时，同意使用MCP咨询工具，获取AI专家的指导。
